1. System Architecture Overview
The KodetoCareer platform follows a modular monolith architecture deployed entirely on free-tier infrastructure
that never expires.
Architecture Pattern
 Backend: Modular Monolith (Node.js / Express.js)
 
 Frontend Web: React.js (Single Page Application)
 
 Mobile: Flutter (iOS + Android, single codebase)
 
 Database: PostgreSQL via Neon.tech (serverless, free)
 
 Cache: Redis via Upstash (serverless, free)
 
 Storage: Cloudflare R2 (S3-compatible, zero egress fees)
 
 CDN: Cloudflare (free, global)
 
 Authentication: JWT (RS256) + Refresh tokens
 
 Real-time: Socket.io

4. Video Architecture (No Transcoding — Free)
Approach for MVP
AWS MediaConvert (paid) is removed. Instead:
Option A — Direct MP4 Upload (for admin-uploaded content):
1. Admin uploads MP4 via presigned R2 URL
 
2. File stored on Cloudflare R2 (up to 10GB free)
 
3. Served directly from Cloudflare CDN — fast delivery globally
 
4. No transcoding — one quality level (original upload quality)
 
5. Video player streams MP4 directly from CDN URL
 
Option B — Embed URL (YouTube/Vimeo — recommended for large videos):
1. Trainer uploads video to YouTube (unlisted) or Vimeo
 
2. Trainer pastes embed URL in admin panel
 
3. Platform embeds the video in an iframe player
 
4. Zero storage cost, unlimited size, CDN handled by YouTube/Vimeo
 
Recommended default: YouTube Unlisted for large course videos. R2 direct upload for short demo/intro clips
under 100MB.
What Changes from Original TRD
 No AWS MediaConvert job creation
 
 No Lambda functions
 
 No HLS transcoding pipeline
 
 No adaptive bitrate (480p/720p/1080p)
 
 Video player uses standard MP4 or YouTube/Vimeo iframe — both work on all devices

7. Email — Resend (Free)
Resend Free Tier: 3,000 emails/month, 100 emails/day — permanently free.
import { Resend } from 'resend';
const resend = new Resend(process.env.RESEND_API_KEY);
export const sendEmail = async ({
  to, subject, html
}: { to: string; subject: string; html: string }) => {
  return resend.emails.send({
    from: 'KodetoCareer <noreply@kodetocareer.com>',

  });

Note on OTP delivery: SMS OTP (MSG91/Twilio) is removed. All OTPs are delivered via email only. This is free
and sufficient for the target user base. WhatsApp OTP can be added in Version 2 via Interakt (free tier available).
8. SMS — Removed from MVP
SMS notifications (MSG91/Twilio) are removed from the MVP.
Replacement: All OTP and critical alerts go via email.
Rationale: Students have email access. Email OTP is reliable. SMS adds ■800–■2,000/month cost with no
proportional benefit for MVP.
Version 2 plan: Add Twilio/MSG91 SMS only when platform is generating revenue.

Oracle Cloud Setup Steps
1. Create Oracle Cloud account (no credit card required for Always Free)
 
2. Provision VM.Standard.A1.Flex: 4 OCPU, 24GB RAM, Ubuntu 22.04
 
3. Configure Security Group: open ports 80, 443, 3000

6. Set up Cloudflare: point domain to Oracle Cloud public IP, enable proxy
 
7. SSL handled automatically by Cloudflare
