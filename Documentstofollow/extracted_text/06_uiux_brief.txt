---
1. Design Philosophy
The KodetoCareer platform must feel professional, trustworthy, and motivating — not like a generic ed-tech
template. Students in tier-2 and tier-3 colleges are comparing this to BYJU's, YouTube, and Instagram. The
design must earn their daily engagement.
Three Design Principles:
1. Progress is Visible at All Times
Every screen should answer "How am I doing?" before the user asks. Progress bars, streak counters, rank
badges, and skill indicators must be persistent and satisfying. When a student completes something, the
interface celebrates it.
2. Clarity Over Cleverness
Every trainer and college admin using the admin panel may not be technically sophisticated. Navigation must be
obvious. Labels must be plain language. No icon-only buttons without labels. Every action must have a
confirmation and a success state.
3. India-First Design
Design for variable connectivity. Skeletons over spinners. Offline-first for previously viewed content. Compress
and lazy-load aggressively. Support right-to-left layouts from day one even if not launched. Respect data
consciousness: show "this video is 45MB" before streaming.
---
2. Brand Identity
2.1 Brand Personality
 Confident — not arrogant. A mentor who believes in the student.
 
 Modern — reflects contemporary tech industry aesthetics, not traditional educational imagery.
 
 Warm — achieves professional tone without feeling cold or corporate.
 
 Achievement-focused — every visual element should imply progress and success.

5. Spacing System
Uses an 8px base grid. All spacing values are multiples of 8 (with 4px as a half-step).
4px    — xs  (icon internal padding, inline elements)
8px    — sm  (component internal spacing)
12px   — md- (list item gap)
16px   — md  (card padding, section gaps)
24px   — lg  (section padding)
32px   — xl  (page section gaps)
48px   — 2xl (major section separators)
64px   — 3xl (hero areas)

6.6 Empty States
Every list, table, or content area must have a designed empty state:
 Illustration (simple line art, consistent style)
 
 Title: "No [items] yet"
 
 Description: what would appear here and why it's empty
 
 Primary action button if applicable
 
Examples:
 No students enrolled: "No students in this batch yet" + [Import Students]
 
 No quizzes created: "No quizzes for this course" + [Create Quiz]
 
 No notifications: "You're all caught up!" + illustration of checkmark
 
6.7 Loading States

 Grey animated gradient rectangles matching the shape of content

 Appears in button while API call is in flight
 
 Button text changes to "Saving..." during load
 
Progress Indicator (for uploads):

9. Motion & Animation
Guiding Principle: Animation confirms state changes, guides attention, and rewards achievement. It should
never slow the user down.
Animation | Duration | Easing | Usage
Screen transition | 250ms | ease-in-out | Route changes
Modal / Bottom sheet enter | 300ms | ease-out | Modal appearance
Modal dismiss | 200ms | ease-in | Modal dismiss
Button press | 100ms | ease | Scale press effect
Toast notification | 200ms enter, 150ms exit | ease | Toast alerts
Skeleton to content | 200ms | ease | Content loaded
Progress bar fill | 600ms | ease-out | On load / update
Confetti / Celebration | 1500ms | custom | Lesson complete, certificate
Achievement animations (Flutter + Lottie):
 Lesson complete: subtle checkmark animation (500ms)
 
 Module complete: medium celebration animation (1000ms)
 
 Course complete: full confetti celebration (2000ms)
 
 Certificate earned: golden glow animation (1500ms)

10. Accessibility
 Colour contrast: All text meets WCAG AA minimum (4.5:1 for body, 3:1 for large text)
 
 Screen reader support: All images have alt text, all icons have labels
 
 Focus management: Keyboard navigation must work on all interactive elements on web
 
 Font scaling: UI must not break at 150% text size on mobile
 
 Error identification: Never use colour alone to indicate errors — always pair with text or icon
